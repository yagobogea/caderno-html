Reproduzindo Áudios em HTML5

Para reproduzir áudios é necessário que o arquivo esteja salvo na mesma pasta do index. Os formatos do áudio podem ser mp3, WAV e OGG, depende do navegador do usuário. Há duas formas de inserir um áudio em um código HTML5. A primeira forma é atráves da tag <audio> e dentro dessa tag é necessário inserir os parâmetros para que o áudio se comporte como desejado. A outra forma de inserir áudios em um site é através também da tag <audio>, mas utilizando essa tag para englobar os <source src="" type=""> que vão dentro. Dessa forma:

Primeira maneira:

<audio src="pasta/arquivo.mp3/wav/ogg">


Segunda maneira:

<audio>
    <source src="pasta/midia.mp3" type="audio/mpeg">
    <source src="pasta/midia.ogg" type="audio/ogg">
    <source src="pasta/midia.wav" type="audio/wav">
</audio>

Também há como inserir parâmetros para o áudio inserido, que são: auto (só vai considerar que o site carregou completamente depois de carregar todo o áudio, isso é perigoso caso o arquivo seja muito grande), metada (carregará o nome do arquivo, duração, data e etc) e none (não carregará nada a não ser que o usuário clique para que isso aconteça). Há também o parâmetro autoplay que fará com que o áudio se inicie automaticamente ao abrir o site, o parâmetro loop para que o áudio se repita ao terminar e o parâmetro controls para que apareça um controle da mídia para o usuário. Caso o navegador do usuário não suporte nenhum tipo de arquivo de áudio, é necessário inserir uma mensagem o avisando e um link para o download. Os parâmetros são inseridos dessa forma:

<audio preload="auto/metadata/none" autoplay controls loop>
    <source src="pasta/midia.mp3" type="audio/mpeg">
    <source src="pasta/midia.ogg" type="audio/ogg">
    <source src="pasta/midia.wav" type="audio/wav">
</audio>

Estilos Inline

Para fazer estilos inline, é necessário usar o parâmetro style="" dentro da tag que se deseja modificar, por exemplo <body style="background-color: pink;"> muda a cor de fundo da página, <h1 style="color: blue;"> muda a cor da fonte h1. Mas como dito em seu nome "estilos inline, a mudança só ocorrerá na tag editada, e as próximas tags iguais a essa, não seguirão esse padrão, logo, será necessário editar todas as desejadas.



Não é recomendado fazer dessa forma, pois isso irá poluir o código HTML5 e criar uma grande confusão. Esse método deve ser usado apenas para pequenas mudanças.

Estilos CSS Internos

Para usar estilos css internos, deve se criar uma tag <style> acima do title, e em seguida criar parâmetros para tags, por exemplo:

body{
    background-color: lightsteelblue;
    font-family: Arial, Helvetica, sans-serif;
    font-size: 20px;
}

Esse código serve para mudar a cor de fundo do body e mudar a fonte e o tamanho de todos os textos que estejam escritos no body. Deve-se usar essa forma para mudar os estilos. Porém há algumas desvantagens nesse tipo de uso de CSS3. A primeira desvantagem é o tamanho de espaço que os styles ocupam, a outra desvantagem é que o códigos dos style não passa de página para página caso eu tanho várias em meu site/na pasta, o que signifique que caso eu queria que todas sigam um padrão, eu necessitaria de copiar o style e colar em cada uma delas, o que é nada prático. Mas para projetos de sites simples, atende bem.

Tipos de cores em um HTML5 com CSS3

Há quatro formas de cores usando css, a primeira é escolher a cor pelo seu nome, a segunda, por RGB, a terceira por Hexadecimal e a quarta por HSL. As cores podem ser inseridas da seguinte forma:

Harmonia de cores

A harmonia de cores em um site é muito importante, pois é necessário saber qual cor combina com outra para fazer um site harmônico/bonito para ser chamativo e leve para o usuário.

Circulo Cromático

O circulo cromático e feito pelas cores rgb, aproximadamente 65 milhões de cores.
Aqui vou separar 12 cores e considerá-las principais para falar sobre harmonia de cores.

Esse circulo cromático é principalmente utilizado para se conseguir harmonizar cores.
Um dos principais pontos para a beleza de um site é a simetria, o outro ponto são as cores. Cores são muito importantes pois podem causar inúmeras sensações no usuário, e a que se deseja principalmente causar, é fazer o usuário irracionalmente achar o site bonito.

Cores primárias

As cores primárias dentro do circulo cromático de 12 cores são: amarelo, vermelho e azul. A harmonia gerada por essas cores não é tão bonita, mas são importantes para podermos representar as outRas cores.

Cores secundárias

As cores secundárias dentro do circulo cromático de 12 cores são: laranja, violeta e verde. A harmonia gerada por essas cores não é tão bonita, meio grosseira.

Cores terciárias

As Cores terciárias são todas as outras que são misturas das cores primárias e secundárias, quando for nomear uma cor terciária, é necessário colocar o nome da cor primária antes e em seguida o nome da cor secundária ex: amarelo-esverdeado, amarelo-alaranjado, vermelho-alaranjado etc. Toda cor dita terciária está entre uma cor primária e uma cor terciária. Essas cores são: amarelo-esverdeado, amarelo-alaranjado, vermelho-alaranjado, vermelho-arroxeado, azul-arroxeado, azul-esverdeado e amarelo-esverdeado.

Temperatura de cores



Uma paleta de cores deve ter de 3 a 5 cores.

Cores complementares

Cores complementares são as cores que tem mais contraste entre si. Para escolher cores complementares, é necessário selecionar uma cor no circulo cromático e em seguida a cor imediatamente oposta a essa cor escolhida primeiro. Ex: Violeta e amarelo, vermelho e verde, azul e laranja, etc. Isso não siginifica que irá ficar bonito, porém para identificar as cores que mais contrastam entre si é dessa forma.

Cores análogas

As cores análogas, ao contrário das cores complementares, são as cores que não tem grande contraste entre si, para identificar cores análogas, é necessário escolher uma cor e pegar as cores imediatamente ao lado dessa cor escolhida primeiro.

Cores análogas relacionadas

Cores análogas relacionadas são feitas para fazer um contraste não tão chamativo, para fazê-las é necessário escolher uma cor principalm pegar uma das cores análogas à ela, pular uma cor e pegar a cor em seguida da cor pulada.

Cores intercaladas

Cores intercaladas tem um princípio parecido com o antrerior, porém tem um contraste maior entre elas.

Cores triádicas

As cores triádicas são cores intercaladas, para fazê-las é necessário escolher uma cor principal, pular três cores e pegar a cor onde cair depois de ter pulado as três, depois pular mais três e fazer o mesmo.

Cores em quadrado

São parecidas com cores triádicas, mas pula-se apenas duas cores ao invés de três e fazer isso três vezes e não duas.

Cores tetrádicas

São conseguidas da seguinte maneira, escolha uma cor qualuer e em seguida ache sua cor complementar, pegar outra cor e achar sua cor complementar novamente.

Monocromia

Trabalha com apenas uma cor, para realizar necessita-se de escolher uma cor e modificar a saturação e o brilho dessa cor.

Criando Gradientes com CSS (degradê)

Para criar um gradiente com CSS, é necessário colocar o parâmetro background-image e escolher a função interna linear-gradient, esse linear-gradient irá indicar a diração do gradiente e quantas cores serão inseridas no mesmo. Dependendo da direção do gradiente, é necessário fazer uma configuração global de tamanho nas CSS, para fazer uma configuração global é necessário criar um seletor global usando *{parâmetro desejado} (* significa all)

Tipografia

Conceitos fundamentais da tipografia.

A tipografia foi inventada quando se deu a necessidade de formar letras que facilitassem a leitura. A palavra tipografia vem de duas palavras gregas: týpos = colocar coisas no papel e graphía = escrita. Sendo assim, tipografio é o estudo de como vou escrever coisas no papel/telas. Escolher as fontes de maneira correta é de suma importância na criação de um site, pois quando um site é bonito e aconchegante visualmente, o usuário fica mais preso ao site. Assim como as cores, as fontes também passam uma emoção ao usuário dependendo da fonte escolhida.

Anatomia dos tipos

Tipo é cada uma das letras, é o modo como vai ser desenhado cada uma das letras. Quando toda fonte é desenvolvida é necessário se preocupar com o tamanho, que é o espaço que ela ocupa. Então, é necessário saber o espaço que a letra vai ocupar ao escolher a fonte. A altura das letras minúsculas de cada fonte é definida a partir do x minúsculo dessa fonte. Já para as letras maiúculas há um tamanho proporcionalmente maior que é chamado de altura das maiúsculas. Outra altura que existe que na grande maioria das vezes é do tamanho das maiúscula, é a altura de vazamento das letras mnúscula, como visto na letra g que vazou para baixo, esse vazamento para baixo é chamado de descendente, já o vazamento para cima como visto nas letras k b, é chamado de altura ascendente. todo dessas alturas é chamado de corpo.

Tamanhos de Fontes

Existem várias formas de se colocar um font-size, há medidas absolutas e as medidas relativas.

    Medidas Absolutas
    • Centímetro (cm)
    • Milímetro (mm)
    • Polegada (in)
    • Pixel (px)
    • Ponto (pt)
    • Paica (pc)

Essas são as medidas absolutas, 1 cm sempre será um cm, mas pode haver um problema na hora de representar isso em diferentes telas, por exemplo, um navegador pode ter dificuldade de representar o tamanho desejado em telas variadas, será diferente na tela de uma tv, diferente na tela de um celular, diferente nas telas de retina, etc.

Modelo de Caixas (Box Model)

O modelo de caixas se aplica tanto ao HTML5 quanto ao CSS3, ele serve para ajudar na organização do design de uma página. A ideia de caixas vem literalmente de caixas reais, e uma das possibilidades de a serem feitas, é inserir caixas dentro de caixas, isso se chama aninhamento. Todo elemento visível de um site é uma caixa, ou seja, quando eu crio um seletor em css para h1 por exemplo, na verdade eu estou estilizando a caixa.
os termos que se referem a modificação de caixa são:

  • height: altura
  • width: largura

Outra coisa que pode ser feita com uma caixa é traçar uma linha em volta do conteudo, em CSS chamado de border, por padrão essa linha ficará grudada na caixa. Para criar um espaço entre a linha traçada e a caixa usa-se a declaração padding (acolchoamento/preenchimento), o padding de uma caixa pode ser definido de certo modo para cado lado ou igualmente para todos os lados.

Um problema que se enfrenta ao criar uma caixa acima da outra, é elas aparecerem na tela coladas uma na outra, para criar um espaço entre essas duas caixas usa-se a declaração margin que faz parecido com o padding, mas ao invés de criar uma distancia entre o border a caixa, criar uma distância externa (configurável) entre as caixas.

Outra área que existe em uma caixa, é a outline (contorno/traçado), que fica colada na border só que por fora da caixa.

Tipos de caixa

Box level e Inline level

Uma caixa box-level ocupará 100%da largura da tela e funcionará tipo como uma área de colisão, fará com que por exemplo, um <p> colocado pule para o seu outro lado.

Uma caixa inline-level será inserida na linha e não fará que o conteúdo seja partido como a caixa acima.

Mudando o tamanho da imagem de fundo

Para mudar o tamanho de uma imagem de fundo usa-se background-size: 100%; | várias medidas podem ser usadas, como px, %, mas é bom colocar 100% pois usando px pode ficar bom em uma tela, porém não em outrasm e o 100% faz com que a imagem de fundo ocupe 100% da tela independente de seu tamanho. Tornando iframes mais seguros

Alguns iframes podem ser perigosos pôr em seu site por questões de roubo de dados, para previnir que um iframe seja capaz de roubar dados estando presente em seu site, basta colocar a declaração "sandbox" com o valor "sandbox" (valor padrão, básico), ficando assim:

<iframe src="#" sandbox="sandbox">

Outros valores para o "sandbox" são:

  • sandbox="allow-same-origin" (faz com que apenas os iframes que sejam da mesma origem/servidor do index possam coletar dados)
  • sandbox="allow-forms" (permite formulários dentro de iframes)
  • sandbox="allow-scripts" (permite scripts dentro dos iframes)

como o sandbox está ativo, essas são as únicas configurações permitidas, todas as outras estão bloqueadas.

não existe apenas essas configurações.

Outra maneira de tornar um iframe mais seguro é utilizando ainda dentro do iframe a declaração "referrerpolicy" com o valor "no-referrer", ficando da seguinte maneira

referrerpolicy="no-referrer".

Isso faz com que o conteúdo que está carregado dentro do iframe não colete dados.

Alguns campos do HTML tem a obrigação de serem preeenchidos antes que os dados sejam enviados. Para fazer o input desejado ter essa obrigação, basta colocar o parâmetro "required" dentro da tag do input desejado. Isso fará que caso o campo não esteja preenchido antes de clicar o botão de submit, apareça um aviso dizendo que este campo tem preenchimento necessário. Caso um campo de texto tenha um número de caracteres mínimo para ser enviado, basta colocar o parâmetro "minlength" e o número de caracteres desejado no campo de escrita à frente do minlength. Para mudar o tamanho de uma caixa de texto, basta colocar o parâmetro "size" dentro do input de caixa desejado e inserir o número desejado dentro da sua área de texto. Os números a serem inseridos são de acordo com os caracteres. Ex: size="10" fará aparecer uma caixa que cabe 10 caracteres. Isso não significa que as letras inseridas a mais não aparecerão, na verdade a caixa começa a ir para o lado junto com a escrita mostrando as letras novas e escondendo as antigas.

Há também o parâmetro "placeholder" que caso tenha um texto escrito dentro da sua área de texto, fará que apareça um texto "fantasma" dentro da caixa de texto.

Navegadores guardam nomes já escritos em caixas de texto e sugerem o nome já escrito anteriormente caso uma nova caixa de texto já clicada, mas como fazemos para o navegador entender que o que está sendo clicado é uma caixa de texto? Para fazer isso, é necessário "dizer" ao input para que cada caixa serve. Colocando o parâmetro "autocomplete" dentro do input, ao fazer isso, aparecerá uma série de sugestões, que dependerá do programador dizer para o input para que cada autocomplete serve. Ex: autocomplete="username" fará que o input entenda que a caixa clicada é uma caixa de nome e sugerirá nomes anteriormente utilizados. Isso tudo em um formulário com o autocomplete="on", claro.

Formulários com telefone e e-mail

Há formulários para email e telefone. Eles são inseridos utilizando os inputs input:email e input:tel. Não são muito úteis pois tem poucas configurações e poucos parâmetros a serem seguidos, porém há como utilizá-los e inserir parâmetros dentro de seus inputs. Um desses parâmetros é uma expressão regular dentro do input de telefone, essa expressão é pattern="^[0-9]{5}-[0-9]{4}$". Toda expressão regular começa com ^ e termina com $, essa expressão posta dentro da área de texto de pattern quer dizer que o input lerá alguns digitos e quantos digitos serão, o input aceitará 5 digitos de 0 a 9 seguido de 4 digitos de 0 a 9 (um número de telefone no padrão brasileiro). Caso seja necessária a adição de ddd entre parênteres o regex fica assim: pattern="^\(\d{2}\)\d{4,5}-\d{4}$"

Essa validação já contida no html5 é ruim, mais para frente será necessário refazer essa validação de dados com Javascript ou PHP.

Elementos color, range e file em HTML

Dentro dos inputs hão as tags color, range e file.

A tag color serve para criar uma caixinha onde o usuário pode escolher uma cor. Caso o dev deseje que esse caixinha venha com uma cor padrão, basta inserir um valor em hexadecimal dentro de um value="".

A tag range cria uma slidebar para que o usuário escolha onde essa barra irá ficar, dentro desse input, é ncessário inserir os valores máximos e mínimos que o usuário poderá escolher, utilizando os parâmetros min e max, caso haja vontade da barra já vir em um certo local, basta utilizar um value com o valor desejado.

A tag file cria um botão que permite enviar arquivos. Para utilizar este input, o form deve estar como post, pois o get suporta apenas 3 mil bytes.

Media Queries

Media Queries são mídias que se adaptam de acordo com o tamanho de telas ou uso.

Há um tamanho da mídia para telas de celulares, monitores, televisões, há inúmeros tamanhos para agregar e adaptar em cada tamanho de tela. Também há media queries para impressão, um site é cheio de links, menus, anúncios e etc, um papel impresso não precisa disso, então, ao clicar no botão de impressão, o site gera um formato apenas para impressão. Media Queries são feitos por CSS.

Dentro dos media queries, há como inserir para onde aquilo está destinado. Por exemplo, dentro do link:css de uma mídia para telas, põe-se o media="screen", impressora media="print", para tudo media="all" e por ai vai. Porém, não há media types para todos os tamanhos de tela, para isso, há os media features. Media queries = media type + media features. Media features são as características de tela.

Para inserir um media feature, igualmente os media types, basta inserir o campo media="" em uma tag de link css e dizer o seu media type, dentro do mesmo campo de media type, coloca-se por exemplo: media="screen and (orientation:landscape)", esse configuração é para indicar se a tela está em retrato ou em landscape, nesse caso landscape. Media features são escritos sempre entre parênteses.
