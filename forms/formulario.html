Meu primeiro formulário

Todo formulário deve estar entre <form> e </form>, dentro dessas tags hão atributos importantíssimos, mas agora aprenderemos apenas o básico. Essa maneira de fazer formulários não é a melhor, mas é utilizavél e utilizaremos ela para aprender.

Fora estar dentro da tag "form", é necessário usar a tag <input> para dar um parâmetro ao form, de como ele irá aparecer na tela e como será utilizado.

No nosso caso, utilizamos o input do type text para as caixas de inserção de nome, as caixinhas de texto foram inseridas automaticamente ao colocar o input. Todo input deve ter um nome e um id.

Também utilizei o input com o type submit para fazer esse botão. O botão foi criado automaticamente sem a necessidade de CSS para fazê-lo. Dentro desse input há a necessidade de insercão de valor "value", inserir esse valor fará com que o texto escrito dentro apareça no botão.

Se você clicar em um dos campos de texto, irá aparecer uma sugestão de autocomplete com nomes utilzados anteriormente em outros sites, se você deseja desativar isso, basta colocar dentro da tag <form> a declaraçao "autocomplete="off"".

Para onde os dados serão enviados?

Normalmente os dados serão enviados para alguma linguagem de programação, há muitas opções, mas aqui utilizarei php apenas como exemplo de um script.

Para fazer isso, basta colocar dentro da tag <form> a declaração "action="cadastro.php"" (.php no meu caso que estou usando a linguagem php como exemplo). Essa ação será disparada quando o botão de submit for clicado.

Textos anteriores às caixas

Antes das caixas de texto, geralmente há palavras, porém o navegador não entende que essa palavra está ligada ao input. Para fazer a ligação entre a palavra que está anexada anteriormente ao input, utiliza-se os "labels", as "etiquetas."

Para utilizar as labels, deve-se envolver a palavra ou texto desejado com a tag <label>, dentro dessa tag irá haver apenas um parâmetro, que é o "for". Dentro do for deve-se inserir o id, isso fará com que o label seja associado ao input, pois o "for" está com o nome do "id", o que significa que os dois estão relacionados.

A parte principal disso é criar uma relação entre o label e o input.

Diferenças entre "name" e "id"

As linguagens de programação precisam pegar o dado de algum lugar, o name e o id ajudam as linguagens a identificarem esse dado. Cada linguagem tem o seu "parâmetro de preferência", por exemplo, a linguagem php utiliza de maneira mais hábil objetos que tenham um name, já a linguagem javascript utiliza melhor o id. Então é necessário identificar todos os campos de entrada.

Métodos de envio de formulários

Quando os dados são enviado para as caixas, o padrão é ficarem expostos na url do site em uso no momento. Isso pode gerar problema de segurança caso seja uma senha ou outra informação privada. Para otimizar a segurança, é necessário utilizar linguagens de programaçao e ter conhecimentos de https. Porém com HTML5 há duas maneiras de prevenir isso, não da melhor forma mas ainda assim útil.

Esses métodos são:

  • GET
  • POST

Esses parâmetros devem ser inseridos dentro da tag <form> utilizando o parâmetro "method="GET"" ou "method="POST"".

O padrão é o GET, que fará com que os dados apareçam na URL como já vimos antes.

Caso o POST seja inserido, ele fará com que os dados não apareçam na URL do site.

Isso não significa que o site está seguro, os dados continuam visíveis, só não na URL, se o inspecionar for utilizado, dá para encontrar os dados facilmente.

Quando utilizar GET e POST?

O GET deve ser utilizado quando os dados não forem sensíveis.

GET faz com que a busca de um site seja compartilhável.

Dados enviados por GET só podem ter até 3 mil bytes (aproximadamente 3 mil letras)

o Get não pode ser utilizado para enviar URLs com foto. Nesses casos é utilizado o post.

O POST deve ser utilizado quando for necessário que o usuário envie para o site dados sensíveism, o ideal é trabalhar com https. Caso você não queira que os dados apareçam na url. Quando os dados passarem de 3 mil bytes. Se for desejado fazer envio de arquivos.
